# Exactly-Once — обзор модуля (KILLER)

Exactly-once семантика — главная гарантия Flink. Реализуется через ABS для internal state + Two-Phase Commit для transactional sinks. Этот модуль детально разбирает 2PC protocol, новый SinkV2 API и production pitfalls (особенно Kafka transaction.timeout.ms).

## E2E exactly-once = 3 слоя

1. **Replayable source** (Kafka offsets, S3 file paths) — можно перечитать
2. **Internal EOS** через ABS — gerantee no record processed twice in state
3. **Transactional sink** через 2PC — gerantee no duplicate written

Если хотя бы один слой at-least-once → итог at-least-once.

## Two-Phase Commit Protocol

```
Phase 1 (Prepare):
  beginTransaction(N)   ← на barrier(N)
  for each record: write to open txn
  preCommit()           ← flush НО не commit; txnId в state

Phase 2 (Commit):
  notifyCheckpointComplete(N)  ← scheduler говорит "checkpoint N committed everywhere"
  commit()                       ← finalize txn

Recovery:
  read pending txns из state → recoverAndCommit()
```

## Critical pitfall: Kafka transaction.timeout

**Сценарий**: Flink restart занимает 30 минут. Kafka transaction.timeout.ms = 15 min (default). Когда Flink восстанавливается, Kafka уже abort-ил pending транзакции → **data loss с EOS**.

**Fix**:
```java
KafkaSink.<String>builder()
    .setProperty("transaction.timeout.ms", "3600000")   // 1 hour
    // также: на broker: transaction.max.timeout.ms = 3600000
```

## SinkV2 API (FLIP-191) — современный contract

Вместо deprecated TwoPhaseCommitSinkFunction:

```java
public interface Sink<InputT> {
    SinkWriter<InputT> createWriter(InitContext context);
}

public interface TwoPhaseCommittingSink<InputT, CommT> extends Sink<InputT> {
    Committer<CommT> createCommitter();
}

// Pre-commit topology (compaction между Writer и Committer):
public interface SupportsPreCommitTopology<...> {
    DataStream<...> addPreCommitTopology(DataStream<...> stream);
}

// Post-commit (Iceberg manifest, Hive metastore):
public interface SupportsPostCommitTopology<...> {
    void addPostCommitTopology(DataStream<...> stream);
}
```

### Iceberg sink использует обе topologies

- **Pre-commit**: small file compaction (объединить мелкие файлы)
- **Commit**: записать data files
- **Post-commit**: publish manifest, update Hive metastore

## Killer takeaway

Если у вас EOS на Kafka и downtime > 15 минут — теряете данные. **ВСЕГДА** установите `transaction.timeout.ms` ≥ предполагаемое max downtime + safety margin. Production setting обычно 1 hour.
