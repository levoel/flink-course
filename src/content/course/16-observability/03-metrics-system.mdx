## –ß–µ—Ç—ã—Ä–µ —Ç–∏–ø–∞ –º–µ—Ç—Ä–∏–∫

### 1. Counter ‚Äî accumulated count

Monotonically increasing integer counter. –ú–æ–∂–Ω–æ increment/decrement, –Ω–æ –æ–±—ã—á–Ω–æ —Ç–æ–ª—å–∫–æ increment.

```java
public class MyMapper extends RichMapFunction<Event, Result> {
    private transient Counter errorCounter;
    private transient Counter processedCounter;

    @Override
    public void open(Configuration cfg) {
        errorCounter = getRuntimeContext()
            .getMetricGroup()
            .counter("errors");

        processedCounter = getRuntimeContext()
            .getMetricGroup()
            .counter("processedRecords");
    }

    @Override
    public Result map(Event e) {
        try {
            Result r = process(e);
            processedCounter.inc();
            return r;
        } catch (Exception ex) {
            errorCounter.inc();
            throw ex;
        }
    }
}
```

Counter –ø–æ–¥—Ö–æ–¥–∏—Ç –¥–ª—è:
- Records processed
- Errors / retries
- Cache hits / misses
- Custom business events

–í Prometheus —ç—Ç–æ **counter** type, –≤ Grafana –æ–±—ã—á–Ω–æ visualized as rate: `rate(my_counter[5m])`.

### 2. Gauge ‚Äî instantaneous value

–¢–µ–∫—É—â–µ–µ –∑–Ω–∞—á–µ–Ω–∏–µ, –º–æ–∂–µ—Ç go up or down. Lambda evaluated –∫–∞–∂–¥—ã–π sample.

```java
public class StatefulMapper extends RichMapFunction<Event, Result> {
    private transient Map<String, Long> userCache = new HashMap<>();

    @Override
    public void open(Configuration cfg) {
        getRuntimeContext().getMetricGroup()
            .gauge("cacheSize", () -> (long) userCache.size());

        getRuntimeContext().getMetricGroup()
            .gauge("memoryUsedMB", () ->
                Runtime.getRuntime().totalMemory() / 1024 / 1024);
    }

    @Override
    public Result map(Event e) {
        userCache.computeIfAbsent(e.getUserId(), k -> 0L);
        return process(e);
    }
}
```

Gauge –ø–æ–¥—Ö–æ–¥–∏—Ç –¥–ª—è:
- Queue depth
- Cache size
- Memory usage
- Connection pool active connections

–í Prometheus —ç—Ç–æ **gauge** type, –≤ Grafana `my_gauge` –Ω–∞–ø—Ä—è–º—É—é.

**Performance**: lambda calculation –≤—ã–∑—ã–≤–∞–µ—Ç—Å—è –Ω–∞ –∫–∞–∂–¥—ã–π scrape (e.g. 15s –≤ Prometheus). –î–æ—Ä–æ–≥–∏–µ calculations ‚Üí noticeable overhead. Cache –ø—Ä–∏ –Ω–µ–æ–±—Ö–æ–¥–∏–º–æ—Å—Ç–∏.

### 3. Meter ‚Äî events per second

Rate measurement. Track records/sec, throughput.

```java
public class MyMapper extends RichMapFunction<Event, Result> {
    private transient Meter recordsPerSecond;

    @Override
    public void open(Configuration cfg) {
        com.codahale.metrics.Meter dropwizardMeter =
            new com.codahale.metrics.Meter();
        recordsPerSecond = getRuntimeContext()
            .getMetricGroup()
            .meter("recordsPerSecond",
                new DropwizardMeterWrapper(dropwizardMeter));
    }

    @Override
    public Result map(Event e) {
        recordsPerSecond.markEvent();   // increment event count
        return process(e);
    }
}
```

Meter computes:
- `getRate()` ‚Äî events/sec
- `getCount()` ‚Äî total events

–≠—Ç–æ convenience wrapper ‚Äî –º–æ–∂–Ω–æ —Ç–æ –∂–µ —Å–∞–º–æ–µ —á–µ—Ä–µ–∑ Counter + Gauge –¥–ª—è rate. Prometheus auto-computes rate –∏–∑ counter.

**–í –±–æ–ª—å—à–∏–Ω—Å—Ç–≤–µ —Å–ª—É—á–∞–µ–≤** ‚Äî –∏—Å–ø–æ–ª—å–∑—É–π—Ç–µ Counter, —Å—á–∏—Ç–∞–π—Ç–µ rate –≤ Prometheus query (`rate(counter[1m])`). Meter –∏–∑–±—ã—Ç–æ—á–µ–Ω –≤ —ç—Ç–æ–π setup.

### 4. Histogram ‚Äî distribution

Distribution measurements: latency, sizes, etc. Compute percentiles (P50, P95, P99).

```java
public class AsyncLookup extends AsyncFunction<Event, Result> {
    private transient Histogram lookupLatency;

    @Override
    public void open(Configuration cfg) {
        com.codahale.metrics.Histogram dwHist =
            new com.codahale.metrics.Histogram(
                new SlidingTimeWindowReservoir(60, TimeUnit.SECONDS));
        lookupLatency = getRuntimeContext()
            .getMetricGroup()
            .histogram("dbLookupLatencyMs",
                new DropwizardHistogramWrapper(dwHist));
    }

    @Override
    public void asyncInvoke(Event input, ResultFuture<Result> resultFuture) {
        long start = System.currentTimeMillis();
        db.lookupAsync(input.getId(), (result, error) -> {
            long elapsed = System.currentTimeMillis() - start;
            lookupLatency.update(elapsed);
            resultFuture.complete(Collections.singleton(result));
        });
    }
}
```

Histogram queries:
- `getStatistics().getQuantile(0.99)` ‚Äî P99
- `getStatistics().getMean()` ‚Äî average
- `getCount()` ‚Äî total samples

**Reservoir matters**: `SlidingTimeWindowReservoir(60s)` ‚Äî rolling 60s window. –ë–µ–∑ —ç—Ç–æ–≥–æ histogram –∏—Å–ø–æ–ª—å–∑—É–µ—Ç `ExponentiallyDecayingReservoir` ‚Äî –±–∏ased –¥–ª—è recent samples –Ω–æ –Ω–µ sharp window.

–í Prometheus histogram type —ç–∫—Å–ø–æ—Ä—Ç–∏—Ä—É–µ—Ç—Å—è –∫–∞–∫ multiple metrics: `_bucket`, `_count`, `_sum`.

## Naming conventions

Flink –Ω–µ enforce conventions, –Ω–æ –µ—Å—Ç—å de-facto standards:

| Metric type | Naming pattern | Example |
|---|---|---|
| Counter | `<noun>` –∏–ª–∏ `<noun>Total` | `processedRecords`, `errorsTotal` |
| Gauge | `<noun>` –ª–∏–±–æ `<noun>Bytes` | `cacheSize`, `memoryUsedBytes` |
| Meter | `<noun>PerSecond` | `recordsPerSecond` |
| Histogram | `<noun>Ms` –ª–∏–±–æ `<noun>Latency` | `dbLookupLatencyMs`, `serializeTimeNs` |

Bad naming:
- `latency` ‚Üí –Ω–µ –ø–æ–Ω—è—Ç–Ω–æ units. Use `latencyMs`.
- `count` ‚Üí counter or gauge? Use `totalCount` (counter) –∏–ª–∏ `currentCount` (gauge).
- `time` ‚Üí units? `timeMs`, `timeNs`.

### Tags vs metric names

Plain names:
```
errors.payment
errors.billing
errors.fraud
```

3 metrics. Cardinality –≤—ã—Å–æ–∫–∞—è, –≥—Ä–∞—Ñ–∏–∫–∏ —Ä–∞–∑–±—Ä–æ—Å–∞–Ω—ã.

Tags approach:
```java
MetricGroup base = getRuntimeContext().getMetricGroup();
base.addGroup("category", category).counter("errors");
```

1 metric `errors` —Å label `category`. –ú–µ–Ω—å—à–µ metrics names, queryable –∫–∞–∫ `sum by (category)`. Prometheus loves —ç—Ç–æ.

## Production patterns

### Pattern 1: Business KPI metrics

```java
public class FraudDetector extends KeyedProcessFunction<String, Event, Alert> {
    private transient Counter fraudDetected;
    private transient Counter falsePositives;
    private transient Histogram detectionLatency;

    @Override
    public void open(Configuration cfg) {
        MetricGroup g = getRuntimeContext().getMetricGroup()
            .addGroup("business");
        fraudDetected = g.counter("fraudDetected");
        falsePositives = g.counter("falsePositives");
        detectionLatency = g.histogram("detectionLatencyMs",
            new DropwizardHistogramWrapper(
                new com.codahale.metrics.Histogram(
                    new SlidingTimeWindowReservoir(60, TimeUnit.SECONDS))));
    }

    @Override
    public void processElement(Event e, Context ctx, Collector<Alert> out) {
        long start = System.currentTimeMillis();
        boolean isFraud = check(e);
        detectionLatency.update(System.currentTimeMillis() - start);
        if (isFraud) {
            fraudDetected.inc();
            out.collect(new Alert(e));
        }
    }
}
```

Business metrics, alerting –Ω–∞ fraud rate spike: `rate(business_fraudDetected[5m]) > 10`.

### Pattern 2: External system health

```java
public class DbLookup extends RichAsyncFunction<Event, Result> {
    private transient Counter dbCallsTotal;

    @Override

        dbCallsTotal = g.counter("callsTotal");
        dbCallsFailures = g.counter("callsFailures");
        dbCallLatency = g.histogram("callLatencyMs", ...);
        g.gauge("connectionsActive", () -> connectionPool.getActive());
    }
    // ...
}
```

Monitor external dependency health: error rate, latency, connection pool depth.

### Pattern 3: State-related debug metrics

```java
public class StatefulProcessor extends KeyedProcessFunction<...> {
    private transient ValueState<UserHistory> state;
    private transient Histogram stateSizeBytes;

    @Override
    public void open(Configuration cfg) {
        state = getRuntimeContext().getState(...);
        stateSizeBytes = getRuntimeContext().getMetricGroup()
            .histogram("stateSizeBytes", ...);
    }

    @Override
    public void processElement(...) {
        UserHistory h = state.value();
        stateSizeBytes.update(h == null ? 0 : h.estimateSize());
        // ...
    }
}
```

Track state size distribution per key ‚Äî detect skew, abnormal users.

## Production gotchas

### Gotcha 1: Histogram reservoir matters

```java
new com.codahale.metrics.Histogram();  // default ExponentiallyDecayingReservoir
```

Default reservoir biased –¥–ª—è recent samples ‚Äî –Ω–æ —ç—Ç–æ –Ω–µ sharp window. –î–ª—è P99 –∑–∞ ¬´last 1 minute¬ª –∏—Å–ø–æ–ª—å–∑—É–π—Ç–µ `SlidingTimeWindowReservoir`.

```java
new com.codahale.metrics.Histogram(
    new SlidingTimeWindowReservoir(60, TimeUnit.SECONDS));
```

### Gotcha 2: Per-key metrics = cardinality explosion

```java
// ANTI-PATTERN:
getMetricGroup()
    .addGroup("user", record.getUserId())   // millions of users
    .counter("events");
```

Million-cardinality label ‚Üí millions of metrics ‚Üí Prometheus dies.

**Fix**: top-N tracking ‚Äî track only top 10 hot users:

```java
private final Map<String, Counter> hotUserCounters = new HashMap<>();
// –ù–∞ –∫–∞–∂–¥—ã–π record: –µ—Å–ª–∏ userId –≤ top-10 (—á–µ—Ä–µ–∑ HyperLogLog –∏–ª–∏ Count-Min), track separately.
```

### Gotcha 3: Gauge lambda –¥–æ—Ä–æ–≥–∞—è

```java
g.gauge("slowMetric", () -> {
    return database.expensiveQuery();   // called every scrape!
});
```

–ö–∞–∂–¥—ã–µ 15s (Prometheus default) —Ç—è–∂—ë–ª–∞—è –æ–ø–µ—Ä–∞—Ü–∏—è ‚Üí blocks metric reporter.

**Fix**: cache value –≤ RichFunction, update lazy:

```java
private volatile long cachedValue = 0;
@Override
public void open(Configuration cfg) {
    // Update periodically —á–µ—Ä–µ–∑ timer
    new Thread(() -> {
        while (running) {
            cachedValue = database.expensiveQuery();
            Thread.sleep(60_000);
        }
    }).start();
    g.gauge("metric", () -> cachedValue);
}
```

### Gotcha 4: Counter overflow

`org.apache.flink.metrics.Counter` –∏—Å–ø–æ–ª—å–∑—É–µ—Ç `long`, overflow –Ω–µ–≤–æ–∑–º–æ–∂–Ω–æ –ø—Ä–∞–∫—Ç–∏—á–µ—Å–∫–∏. –ù–æ –µ—Å–ª–∏ –∏—Å–ø–æ–ª—å–∑—É–µ—Ç–µ `int` counter from Dropwizard —á–µ—Ä–µ–∑ wrapper ‚Äî overflow –≤–æ–∑–º–æ–∂–µ–Ω.

### Gotcha 5: Metric name conflicts

Two operators named same ‚Üí metrics merge with same name. Confusion –≤ Grafana.

**Fix**: explicit operator names:

```java
.uid("my-mapper")
.name("MyMapper")
```

`name()` –∏–¥—ë—Ç –≤ metric path ‚Üí unique names = unique metrics.

### Gotcha 6: Reporter overhead

Prometheus reporter scraping ‚Üí CPU/network spike –∫–∞–∂–¥—ã–µ 15s. –ù–∞ huge cluster —ç—Ç–æ noticeable.

**Fix**: filter unused metrics:

```yaml
metrics.reporter.prom.scope-filter: "!flink.taskmanager.*.shuffle.*"
```

Exclude expensive subsystems –≤—Ä–æ–¥–µ network shuffle internals.

### Gotcha 7: Time units inconsistent

Built-in metrics use mix of nanoseconds, milliseconds, seconds. `currentInputWatermark` ‚Äî Unix milliseconds. `checkpointStartDelayNanos` ‚Äî nanoseconds. Verify units –ø–µ—Ä–µ–¥ alerting!

## Diagnostic checklist –¥–ª—è metrics

1. **Are metrics being exported?** Prometheus targets discoverable?
2. **Counter rate**: `rate(counter[5m])` non-zero?
3. **Histogram percentiles**: P50/P95/P99 reasonable?
4. **Gauge fluctuation**: stable plateau –∏–ª–∏ –ª–∏–Ω–µ–π–Ω—ã–π —Ä–æ—Å—Ç?
5. **Cardinality**: metric explosion?
6. **Naming consistency**: easy filter –∏ query?
