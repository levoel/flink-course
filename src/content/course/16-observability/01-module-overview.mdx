## Top 12 metrics для production dashboard

1. `lastCheckpointDuration` p50/p95/p99 — stability indicator
2. `lastCheckpointSize` — state growth trend
3. `numberOfFailedCheckpoints` (counter) — alert > 0
4. `currentInputWatermark` per operator — stuck watermark detection
5. `numLateRecordsDropped` (если без side output)
6. `Status.JVM.GarbageCollector.G1Young.Time` — GC pressure
7. `Status.JVM.Memory.Direct.MemoryUsed` — off-heap budget
8. `taskmanager.Status.Network.AvailableMemorySegments` — network buffer health
9. `busyTimeMsPerSecond` per operator
10. `backPressuredTimeMsPerSecond` per operator
11. `numRecordsInPerSecond` / `numRecordsOutPerSecond` — throughput
12. RocksDB compaction count + duration

## OpenTelemetry в Flink 2.3

gRPC export (FLIP-???):

```yaml
metrics.reporter.otel.factory.class: org.apache.flink.metrics.otel.OtelReporterFactory
metrics.reporter.otel.endpoint: http://otel-collector:4317
metrics.reporter.otel.protocol: grpc
traces.reporter.otel.endpoint: http://otel-collector:4317
```

Distributed traces показывают full lifecycle checkpoint: barrier insertion → propagation → ack → notify complete. Critical для debugging slow checkpoints.

## Alerting playbook

| Metric | Alert | Severity |
|---|---|---|
| `lastCheckpointDuration` > timeout/2 | Warning | Medium |
| `numberOfFailedCheckpoints` > 0 (5min window) | Alert | High |
| `currentInputWatermark` lag > 5min | Alert | High |
| `numLateRecordsDropped` > 0 | Alert | Medium |
| `backPressuredTimeMsPerSecond` > 800 (sustained 20s) | Alert | High |
| TaskManager OOMKilled | Alert | Critical |

## Killer takeaway

**Backpressure visualization в Web UI** — главный диагностический инструмент. Изучите его до автоматизма. 80% production issues видны через busy/idle/backpressure pattern в первую минуту анализа.
