# Обзор курса и дорожная карта

Этот курс — самое глубокое погружение в Apache Flink на русском языке. Цель — не «научиться писать stream job», а **понять, как Flink работает под капотом**: как Chandy-Lamport snapshots реализованы через ABS алгоритм, как RocksDB LSM-tree интегрирован с managed memory, как Two-Phase Commit обеспечивает exactly-once, как ForSt позволяет хранить state на S3 без потери latency.

---

## Целевая аудитория

Курс рассчитан на инженеров, которые:

- Знают Kafka на уровне «писал producer/consumer, понимают partitions и consumer groups»
- Имеют опыт работы с Apache Spark (Structured Streaming желательно, но не обязательно)
- Знают Java на уровне написания production-кода (Scala API удалён в Flink 2.0 — общий ML/programming опыт OK); Python для PyFlink модуля
- Хотят строить low-latency stateful pipelines и понимать их fundamentals

Если вы впервые видите streaming — рекомендуем сначала пройти kafka-course и spark-course с этой же платформы. Этот курс — продолжение, не первый шаг.

## Что вы получите

К концу курса вы:

1. **Разверните production HA-кластер Flink** на K8s через Kubernetes Operator с Adaptive Scheduler.
2. **Спроектируете exactly-once pipeline**: Kafka → Flink stateful processing → Iceberg, end-to-end EOS гарантия.
3. **Затюните RocksDB** для своего workload — compaction strategy, column families, managed memory.
4. **Диагностируете backpressure** через busy/idle/backPressured метрики и Web UI визуализацию.
5. **Мигрируете state** через State Processor API при изменении topology или backend.
6. **Внедрите Flink CDC** для real-time replication из Postgres в lakehouse.
7. **Аргументируете выбор Flink vs Spark Structured Streaming vs Kafka Streams vs RisingWave**.

## Формат уроков

Каждый модуль состоит из 5-7 уроков:

- **MDX-урок** — теория с диаграммами, кодом, callout-блоками. 20-40 минут чтения.
- **Knowledge check** — короткая проверка ключевого понимания.
- **Quiz** — 4 вопроса (multiple-choice + true/false), bloomLevel: conceptual / applied / analytical / evaluative.
- **Module exam** — 6 вопросов, passing threshold 70%.
- **Docker lab** — на ключевые модули: barrier propagation tracing, RocksDB compaction live, disaggregated state с MinIO.
- **Glossary** + **Troubleshooting** — справочные ресурсы для модулей.

## Стек курса

- **Apache Flink 2.2.x** (current stable, 2.0 как база)
- **Java 17 / 21** — основной язык
- **Python 3.11** — для PyFlink модуля
- **Apache Kafka 3.7** — source/sink для большинства labs
- **Apache Paimon** — lakehouse backend для Materialized Tables
- **MinIO** — S3-compatible storage для disaggregated state labs
- **Flink Kubernetes Operator** + **OrbStack/Kind** — для K8s deployment модуля
- **Flink CDC 3.6** — для real-time replication модуля

---

## Что НЕ покрываем (legacy)

Намеренно опускаем то, что removed/deprecated в 2.x:

- **DataSet API** — removed в 2.0
- **Scala API** — removed в 2.0
- **SourceFunction / SinkFunction / SinkV1** — removed в 2.0
- **Per-job mode** — removed в 2.0
- **flink-conf.yaml** (legacy format) — заменён на standard YAML
- **Stateful Functions** — sunsetted в 2026
- **Queryable State** — deprecated с 1.18
- **Reactive Mode** — заменён Adaptive Scheduler + Autoscaler
