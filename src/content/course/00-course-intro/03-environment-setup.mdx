# Окружение и first job

Этот урок — практический. К концу у вас будет работающий Flink 2.2 в Docker, запущенный stateful job, понимание Web UI и базовых CLI команд. Дальнейшие модули предполагают, что окружение поднято.

---

## Требования

- **Docker Desktop / OrbStack** — для macOS лучше OrbStack
- **Минимум 6 ГБ свободной оперативной памяти** (Flink JVM + Kafka + Zookeeper-less KRaft)
- **Java 17 или 21** — для компиляции примеров локально (`brew install openjdk@21`)
- **Maven 3.9+** или **Gradle 8** — для сборки проектов

<Callout type="tip">
Если вы на ARM Mac (M1/M2/M3), используйте images с тегом `2.2.0-java17` — они multi-arch и работают нативно. Старые `flink:1.x` могут стартовать через эмуляцию x86_64 и быть в 3-5 раз медленнее.
</Callout>

## docker-compose.yml для Flink

Создайте `flink-course-env/docker-compose.yml`:

```yaml
services:
  jobmanager:
    image: flink:2.2.0-java17
    ports:
      - "8081:8081"
    command: jobmanager
    environment:
      - |
        FLINK_PROPERTIES=
        jobmanager.rpc.address: jobmanager
        state.backend.type: rocksdb
        state.checkpoints.dir: file:///tmp/flink-checkpoints
        state.savepoints.dir: file:///tmp/flink-savepoints
        execution.checkpointing.interval: 60s
        execution.checkpointing.mode: EXACTLY_ONCE
        taskmanager.numberOfTaskSlots: 4

  taskmanager:
    image: flink:2.2.0-java17
    depends_on:
      - jobmanager
    command: taskmanager
    scale: 2
    environment:
      - |
        FLINK_PROPERTIES=
        jobmanager.rpc.address: jobmanager
        taskmanager.numberOfTaskSlots: 4
        taskmanager.memory.process.size: 2048m
        state.backend.type: rocksdb

  kafka:
    image: apache/kafka:3.7.0
    ports:
      - "9092:9092"
    environment:
      KAFKA_NODE_ID: 1
      KAFKA_PROCESS_ROLES: broker,controller
      KAFKA_LISTENERS: PLAINTEXT://:9092,CONTROLLER://:9093
      KAFKA_ADVERTISED_LISTENERS: PLAINTEXT://kafka:9092
      KAFKA_CONTROLLER_LISTENER_NAMES: CONTROLLER
      KAFKA_LISTENER_SECURITY_PROTOCOL_MAP: PLAINTEXT:PLAINTEXT,CONTROLLER:PLAINTEXT
      KAFKA_CONTROLLER_QUORUM_VOTERS: 1@kafka:9093
      KAFKA_OFFSETS_TOPIC_REPLICATION_FACTOR: 1
```

Запустите:

```bash
docker compose up -d
docker compose ps      # должны быть jobmanager, taskmanager (×2), kafka
```

Откройте `http://localhost:8081` — увидите Flink Web UI с пустым cluster: 1 JobManager, 2 TaskManager-а, 8 slots всего (4 на TM × 2 TM).

## Первый job: WordCount

Создайте Maven проект:

```bash
mvn archetype:generate \
    -DarchetypeGroupId=org.apache.flink \
    -DarchetypeArtifactId=flink-quickstart-java \
    -DarchetypeVersion=2.2.0 \
    -DgroupId=com.example \
    -DartifactId=wordcount \
    -Dversion=1.0
```

Замените `src/main/java/com/example/DataStreamJob.java`:

```java
package com.example;

import org.apache.flink.api.common.eventtime.WatermarkStrategy;
import org.apache.flink.api.common.functions.FlatMapFunction;
import org.apache.flink.api.common.serialization.SimpleStringSchema;
import org.apache.flink.api.java.tuple.Tuple2;
import org.apache.flink.connector.kafka.source.KafkaSource;
import org.apache.flink.connector.kafka.source.enumerator.initializer.OffsetsInitializer;
import org.apache.flink.streaming.api.datastream.DataStream;
import org.apache.flink.streaming.api.environment.StreamExecutionEnvironment;
import org.apache.flink.util.Collector;

public class DataStreamJob {
    public static void main(String[] args) throws Exception {
        StreamExecutionEnvironment env = StreamExecutionEnvironment.getExecutionEnvironment();
        env.enableCheckpointing(60_000);

        KafkaSource<String> source = KafkaSource.<String>builder()
                .setBootstrapServers("kafka:9092")
                .setTopics("words")
                .setGroupId("wordcount")
                .setStartingOffsets(OffsetsInitializer.earliest())
                .setValueOnlyDeserializer(new SimpleStringSchema())
                .build();

        DataStream<String> lines = env.fromSource(
                source, WatermarkStrategy.noWatermarks(), "kafka-source");

        DataStream<Tuple2<String, Integer>> counts = lines
                .flatMap((FlatMapFunction<String, Tuple2<String, Integer>>) (line, out) -> {
                    for (String word : line.split("\\W+")) {
                        if (!word.isEmpty()) out.collect(Tuple2.of(word, 1));
                    }
                })
                .returns(org.apache.flink.api.common.typeinfo.Types.TUPLE(
                        org.apache.flink.api.common.typeinfo.Types.STRING,
                        org.apache.flink.api.common.typeinfo.Types.INT))
                .keyBy(t -> t.f0)
                .sum(1);

        counts.print();

        env.execute("WordCount");
    }
}
```

Соберите и запустите:

```bash
mvn clean package
docker cp target/wordcount-1.0.jar $(docker compose ps -q jobmanager):/opt/flink/wordcount.jar
docker compose exec jobmanager flink run -d /opt/flink/wordcount.jar
```

В UI на `:8081` увидите Running job. Создайте topic и отправьте слова:

```bash
docker compose exec kafka kafka-topics.sh \
    --bootstrap-server kafka:9092 --create --topic words

docker compose exec kafka kafka-console-producer.sh \
    --bootstrap-server kafka:9092 --topic words
> hello flink world
> flink stream processing
```

В UI кликните на job → Subtasks → один из subtasks → View Logs — увидите вывод `(flink, 2)`, `(hello, 1)` и так далее.

## Базовый CLI

Все важные операции — через `flink` CLI внутри JobManager:

```bash
# Список running jobs
docker compose exec jobmanager flink list

# Запуск jar
docker compose exec jobmanager flink run -d /opt/flink/wordcount.jar

# Остановка job с savepoint
docker compose exec jobmanager flink stop --savepointPath file:///tmp/flink-savepoints <jobid>

# Cancel без savepoint
docker compose exec jobmanager flink cancel <jobid>

# Restart job с savepoint
docker compose exec jobmanager flink run -s file:///tmp/flink-savepoints/<savepoint-name> /opt/flink/wordcount.jar

# Получить savepoint статус
docker compose exec jobmanager flink savepoint <jobid> file:///tmp/flink-savepoints
```

В модуле 06 мы детально разберём чем savepoint отличается от checkpoint (canonical vs native format, ownership, lifecycle).

## Web UI: что смотреть

Откройте `:8081` и кликните на ваш job. Ключевые экраны:

1. **Overview** — общее состояние job, parallelism, slots usage.
2. **Subtasks** — каждый subtask с метриками bytes/records sent/received, status. Здесь видно неравномерное распределение нагрузки (skew).
3. **Watermarks** — текущий watermark каждого оператора. Полезно для отладки stuck watermarks.
4. **Checkpoints** — история checkpoint-ов, последний duration, размер, ack status.
5. **Backpressure** — для каждого оператора показывает `busyTimeMsPerSecond` / `idleTimeMsPerSecond` / `backPressuredTimeMsPerSecond`. Цвета: красный=busy, синий=idle, чёрный=backpressured.

<Callout type="tip">
Метрика **backpressure** — первый инструмент диагностики любой Flink job. Если оператор имеет `backPressuredTimeMsPerSecond > 0` — downstream не успевает обрабатывать его output. Это часто значит, что sink медленный, или один из downstream operators имеет skewed key distribution.
</Callout>
