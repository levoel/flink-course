# PyFlink — обзор модуля

PyFlink даёт Python API для DataStream и Table. Использовать когда команда строго Python (ML инженеры), но понимать perf trade-offs vs JVM. Этот модуль — practical guide к PyFlink с perf hierarchy.

## PyFlink performance hierarchy

```
Native Flink Java         100% baseline
Python Thread mode         70-90% (PEMJA, JVM-thread)
Python Process mode         30-50% (Beam Portability, IPC)
Python Process + custom deps  20-40%
Pandas UDF (vectorized)   90-95% (Arrow zero-copy, batch)
```

## Pandas UDFs — game changer

Vectorized via Apache Arrow:

```python
from pyflink.table.udf import udf
import pandas as pd

@udf(input_types=[DataTypes.DOUBLE()], result_type=DataTypes.DOUBLE())
def double_value(v):
    return v * 2  # row-by-row, slow

@udf(input_types=[DataTypes.DOUBLE()], result_type=DataTypes.DOUBLE(),
     func_type='pandas')   # ← VECTORIZED
def double_pandas(v: pd.Series) -> pd.Series:
    return v * 2  # batch операция, 10-50x быстрее
```

## Async functions (Flink 2.2)

Для external service calls (LLM, embedding APIs):

```python
@async_udf(input_types=[DataTypes.STRING()], result_type=DataTypes.STRING())
async def llm_call(text: str) -> str:
    response = await openai.ChatCompletion.acreate(
        model="gpt-4", messages=[...]
    )
    return response.choices[0].message.content
```

## Когда НЕ использовать PyFlink

- Stateful keyed jobs с complex state primitives — лучше Java
- Latency-critical (&lt; 50ms) — IPC overhead dominates
- High throughput (> 100k rec/s per slot) — Java native parsing
- Custom serializer / format work — Java API богаче

## Killer takeaway

Если ваша job всё равно ML-inference heavy — PyFlink с Pandas UDFs или async functions может быть быстрее, чем Java + Async I/O к external Python service. Embedded execution убирает network hop.
