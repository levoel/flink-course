## Ключевые концепты

### JobManager — три компонента

- **Dispatcher** — REST API, принимает job submissions, стартует JobMaster per job
- **ResourceManager** — один на cluster, управляет slots, на K8s native общается с K8s API
- **JobMaster** — один на job, отвечает за life cycle, держит ExecutionGraph

### Graph transformation

```
StreamGraph (API calls)
  ↓ operator chaining optimization
JobGraph (tasks с chained operators)
  ↓ parallelism expansion
ExecutionGraph (N ExecutionVertex per task)
  ↓ slot allocation
Physical execution (deployed на slots)
```

### Slot Sharing Groups

By default все операторы job в slot-sharing-group `"default"`. Это позволяет N параллельных копий полного pipeline-а уместиться в N slots. Изолировать state-heavy operator:

```java
stream.keyBy(...).process(heavyOp).slotSharingGroup("heavy-state");
```

### Operator Chaining

Соседние операторы со совместимым parallelism и forward partitioning склеиваются в один task → одна нить, ноль сериализации. Но ломает independent scaling — disable если operators имеют разные resource needs.

## Killer takeaway

Slot — не CPU и не memory. Slot — это **единица планирования**. CPU shared (soft isolation), memory разделена поровну между slots. Это фундаментальное отличие от Spark executor model.
