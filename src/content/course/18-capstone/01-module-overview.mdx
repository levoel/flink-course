## Capstone architecture

```
Postgres (OLTP)
    ↓ Debezium binlog
Kafka (orders, customers, products topics)
    ↓ Flink CDC 3.6
        ├──→ Flink stateful job
        │       (per-user aggregations,
        │        temporal joins,
        │        watermark alignment)
        ↓
    Iceberg (lakehouse, EOS sink)
    ↓
    ClickHouse (real-time OLAP)
    ↓
    Grafana dashboards
```

## Что демонстрируется в Capstone

- ✅ **State Management** (M3) — ValueState, MapState, TTL
- ✅ **RocksDB + ForSt** (M4) — disaggregated state на S3
- ✅ **Watermark Alignment** (M5) — multi-source coordination
- ✅ **Generic Log Incremental Checkpoints** (M6) — stable checkpoint latency
- ✅ **Two-Phase Commit** (M7) — exactly-once на Iceberg sink
- ✅ **Materialized Tables** (M8) — aggregated metrics для dashboards
- ✅ **Flink CDC 3.6** (M10) — source pipeline
- ✅ **Flink Kubernetes Operator + Adaptive Scheduler** (M13)
- ✅ **State Processor API** (M14) — для bootstrap state
- ✅ **OpenTelemetry tracing** (M16) — production observability

## Chaos testing checklist

- [ ] `docker kill TaskManager` → job рестартует с savepoint, no data loss
- [ ] `docker kill JobManager` → leader election, new JM pickups
- [ ] Network partition Kafka ↔ Flink → backpressure handled
- [ ] Kafka broker down → consumer reassign partitions
- [ ] Postgres source down → CDC pipeline graceful pause
- [ ] S3/Iceberg slow → unaligned checkpoints kick in
- [ ] OOM kill TaskManager → adaptive scheduler reschedules

## Production readiness checklist

- [ ] Все stateful operators имеют UID
- [ ] Watermark alignment configured если multi-source
- [ ] Side output для late events
- [ ] Allowed lateness reasonable
- [ ] State TTL configured
- [ ] Changelog state backend enabled
- [ ] Unaligned checkpoint timeout configured
- [ ] Transaction timeout >= max downtime
- [ ] Adaptive scheduler enabled
- [ ] Autoscaler thresholds tuned
- [ ] Grafana dashboard + alerts
- [ ] Savepoint test recovery procedure documented
- [ ] Disaster recovery runbook
